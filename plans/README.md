## Recommended order for 004-006

1. `004` first: S-effort, zero-risk, and it fixes the agent-facing feature map
   that every later task relies on for context.
2. `005` and `006` are independent spikes. Recommendation: `005` for the bigger
   AI value, `006` for the bigger draft-safety win.

## Recommended order for 007-010

1. `007` first: it prevents duplicate AI calls in the thread summary flow.
2. `008` next: it fixes a page-lifecycle bug that can weaken self-user checks.
3. `009` next: it reduces unnecessary full-storage reads from the Mobile Lite
   settings UI.
4. `010` last or in parallel with the others: it is independent, but it makes
   the registry safer before more Mobile Lite modules are added.

- `002` and `003` depend on `001` because lifecycle cleanup should be reliable
  before adding more async failure handling around user-card actions and dialogs.
- `004`, `005`, and `006` have no hard dependencies on each other. `005` and
  `006` are design/spike plans: each produces design notes plus a minimal
  flag-gated prototype, not a finished feature, and should spawn a follow-up
  build plan once its decisions are settled.
- `007`, `008`, `009`, and `010` are independent from each other and from
  `001`-`006`.
- `010` is a useful prerequisite before future module-heavy Mobile Lite work,
  but it is not a hard dependency for the other plans.
